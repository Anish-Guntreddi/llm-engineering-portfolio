Architecture & recipe

Self-validating data: parametric generators produce candidates; every output is validated against the schema before acceptance, then a stratified leak-free split (360/48/72). 480 examples across 8 workflow categories.

Training: QLoRA — 4-bit NF4, LoRA r=16/α=32 on attention + MLP, 3 epochs, lr 2e-4 cosine, completion-only loss masking, paged 8-bit AdamW, fixed seed.

The base model already emits valid JSON (0.97). The win is schema consistency — correct category, valid trigger, only declared systems — exactly what a small model must learn.

The sandbox (the differentiator)

Only code that clears a static import allow-list is ever executed — in an isolated subprocess with a hard timeout, reduced builtins, a guarded importer and a meta_path blocker. A str.format dunder-walk escape was found and closed.

Honest boundary: a pure-Python sandbox isn't a hard wall — the real boundary is refusing to execute dangerous code at all (os/socket/subprocess/open denied statically), plus subprocess + timeout. Documented with a containerize-the-child fix path.

Fine-tuned output is also tighter: latency 11.7 s → 5.5 s.

Architecture & security

Pipeline: ingest (PDF/MD/TXT, page-tagged chunks) → embed → pgvector → top-k cosine over the user's chunks → grounded answer + citations → abstain when confidence is low.

Audited (/cso): IDOR (row-level user scoping, verified against the live DB), malicious uploads, prompt injection (instruction boundary), and key leakage (Gemini key server-side only).

The finding: both adaptation methods close ~the same gap from base — fine-tuning +0.201 composite, few-shot RAG +0.200. For this structured task at 1.5B, in-context examples are about as effective as a weight update — and the fairness controls (RAG retrieves only from the train split, the same knowledge the fine-tune saw) are what make that comparison trustworthy. Fine-tuning is also the fastest (4.9 s) and best on field-level metrics; the hybrid wins overall only by a hair (+0.011), almost entirely from the lowest hallucination (0.01). Verified deterministic: a cache-cleared rerun reproduced the numbers byte-identically.
03

Methodology

How the models were trained and why the numbers are trustworthy.

How the LLMs were trained

  • Base models: Qwen2.5-1.5B-Instruct & Qwen2.5-Coder-1.5B-Instruct.
  • Quantization: 4-bit NF4, double quant, bf16 compute.
  • LoRA: r=16, α=32, dropout 0.05, on q/k/v/o + gate/up/down.
  • Optimization: 3 epochs, lr 2e-4 cosine, paged 8-bit AdamW, completion-only loss, fixed seed.
  • Hardware: a single RTX 4090 (24 GB) — every run fits with room to spare.

Why the numbers hold

  • Pinned dependencies, fixed seeds, versioned configs per run.
  • The validator/scorer is a single source of truth across data, eval, and serving.
  • Self-validating datasets — bad examples rejected before training.
  • Strict train/val/test discipline; metrics computed identically across compared systems.
  • Results reported honestly (CSV + markdown), including failures and caveats.
